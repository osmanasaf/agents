# Refactor Pipeline (Manual)

1. Run context collection to capture stack, architecture, and existing rules.
2. Convert the refactor goal into requirements with acceptance criteria.
3. Aggregate applicable rules into a single active ruleset.
4. Create a phased plan (planner).
5. Break phases into concrete tasks with dependencies.
6. Validate coverage against requirements and rules.
7. Implement changes in small increments.
8. Review changes (reviewer) for SOLID, Clean Code, security, performance.
9. Debug regressions (debugger) and add missing tests.
